# Pineapple Party

## What this is

Single-file Three.js app (`index.html`) for viewing and deforming stylized 3D fruit models, then exporting them as OpenSCAD `.scad` files. No build step, no dependencies beyond CDN-loaded Three.js.

## Architecture

- **`models/*.scad`**: 7 SCAD parameter files (one per fruit). Each has `key = value;` parameter assignments at the top (parsed by JS) and valid standalone OpenSCAD modules below. These are the single source of truth for fruit geometry.
- **`index.html`**: The viewer app. Key sections:
  - `parseScadParams(text)` — regex parser for `key = value;` lines from SCAD files
  - `buildFruitFromParams(p)` — generic builder that creates a Three.js model from parsed params. Dispatches on `body_type` (sphere/lathe/tube/extrude/star), `bump_placement` (grid/curve/random/ridge), and `leaf_placement` (crown/surface).
  - Async initialization fetches all 7 `models/*.scad` files and builds models via `Promise.all`.
  - **Sliders**: 12 animation/deformation sliders in `state` object (extrude, squeeze, roll, smush, galumph, throb, wibble, splay, melt, sulpherate, disco, invert). Applied every frame in the animation loop.
  - **Deform tools**: Grab, inflate, pinch, spike — modify `bump.deformOffset` vectors. Body mesh vertices are only modified by the invert slider.
  - **SCAD export**: Snapshots the current rendered state. Body exported as `polyhedron()` from live vertex positions. Bumps/leaves read from `mesh.position` and `mesh.scale` to capture all slider + deform effects. Dispatches on `m.bumpType` and `m.leafPlacement` (not model name).
  - **Animation**: `requestAnimationFrame` loop applies all slider effects to bumps, leaves, crown, and body each frame. Timeline scrubber controls `animTime`.

## Running

```
python3 -m http.server 8765
```

Or use the Claude preview server config in `.claude/launch.json`.

## Coordinate systems

- Three.js: Y-up (x right, y up, z toward camera)
- OpenSCAD: Z-up (x right, y forward, z up)
- SCAD export swaps Y↔Z and reverses face winding to fix handedness

## Key invariants

- Bump positions in SCAD export come from `b.mesh.position` (the actual rendered position), not `origPos + deformOffset`. This ensures all slider effects are captured.
- Body polyhedron faces have reversed winding order (`[i0, i2, i1]` instead of `[i0, i1, i2]`) because the Y↔Z swap flips handedness.
- Each model's `origColors` stores the original Three.js Color objects for SCAD color export.
- `pineapple.scad` (root) is a legacy reference file, not generated by the app.
- Model objects carry `bumpType`, `bumpPlacement`, `leafPlacement`, and `params` for SCAD export dispatch (no name-based conditionals).
